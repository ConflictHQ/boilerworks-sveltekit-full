# Boilerworks SvelteKit Full -- Bootstrap

## Conventions

1. **UUID primary keys** -- All tables use `uuid` PKs generated by `gen_random_uuid()`. Never expose integer IDs.
2. **Soft deletes** -- Set `deleted_at` and `deleted_by`. Never hard delete. All queries filter `WHERE deleted_at IS NULL`.
3. **Audit trails** -- All base models have `created_at`, `updated_at`, `created_by`, `updated_by`.
4. **Permission checks** -- Every `load()` and form action calls `requirePermission()` or `requireAuth()`.
5. **Svelte 5 runes** -- Use `$props`, `$state`, `$derived`. No legacy `export let` or stores.
6. **Form actions** -- Use SvelteKit form actions for mutations. Return `fail()` for validation errors.
7. **Server-only imports** -- Database, auth, and permission logic lives in `$lib/server/`. Never import in client code.
8. **Tailwind dark theme** -- Use `bw-*` custom colors. No inline styles.

## Database

- Drizzle ORM with PostgreSQL (postgres.js driver)
- Schema in `src/lib/server/db/schema.ts`
- Migrations via `drizzle-kit push` (dev) or `drizzle-kit generate` (production)
- Seed via `npm run db:seed`

## Auth

- Session tokens stored as SHA-256 hashes in `sessions` table
- Passwords hashed with argon2 via `@node-rs/argon2`
- Session validated in `hooks.server.ts` on every request
- 30-day session expiry

## Permissions

- Users belong to Groups (M2M via `user_groups`)
- Groups have Permissions (M2M via `group_permissions`)
- Superusers bypass all permission checks
- Use `requirePermission(locals.user, 'resource.action')` pattern

## Testing

- Unit tests: `tests/unit/` -- Vitest
- E2E tests: `tests/e2e/` -- Playwright
- Run: `npm run test` / `npm run test:e2e`
